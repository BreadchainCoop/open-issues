The scope is well-defined with 6 implementation phases that fit perfectly in 2-3 weeks.

stacks (saving-circles) #122

Fee-on-Transfer / Rebasing ERC20s Break Deposit Accounting

Problem

  • Deposits credited with requested value, not actual received
  • Fee-on-transfer tokens cause under-collateralization
  • Cross-circle fund contamination possible
  • Can cause insolvency/reverts locking user funds

Deliverables

  • Normalize deposit accounting to actual received
  • balanceAfter - balanceBefore pattern implementation
  • Policy documentation for token allowlisting
  • Per-circle escrow isolation consideration
  • Comprehensive test coverage

Why This Issue?

This is a high-severity security bug with clear remediation steps - perfect for learning vulnerability analysis. The issue demonstrates a classic ERC20 edge case that has caused real losses in DeFi protocols. You'll learn defensive programming patterns, understand token transfer mechanics deeply, and gain experience writing security-critical code. The proof of concept and affected code locations are clearly documented, making this accessible yet impactful.

Security ERC20 Solidity DeFi Vulnerability Fix
safety-net #14

Change Contestation Threshold Logic for Withdrawal Approval

Current Behavior

  • Any contest triggers voting phase
  • Requires all members to vote
  • 66% yes votes needed for approval
  • Can stall on low participation

Proposed Change

  • Auto-approve unless >33% contest
  • No voting phase needed
  • Simpler, faster withdrawals
  • Minority can still block bad actors

Why This Issue?

This is a governance mechanism redesign that teaches you about on-chain voting systems. The issue includes before/after Mermaid sequence diagrams showing the exact flow changes. You'll implement threshold-based approval logic, handle edge cases like abstentions and member churn, and consider Sybil attack resistance. This is practical experience with DAO governance patterns used in major protocols.

Governance Solidity Voting Systems Game Theory Security
